# Admin Review Canon

## Core Rule

Automation handles intake. Admins handle authority.

## Review Queues

Admins review:

- Canon Change Requests
- Memory Review Queue
- Decision Candidates
- Sensitive Review Items
- Access Failures
- CCOS Ledger Drafts
- Unassigned Tasks
- High Severity Risks
- Processing Errors

## Canon Change Requests

Created when content affects:

- Governing purpose
- Policies
- Circle definitions
- Role definitions
- Decision rights
- Legal commitments
- Financial commitments
- Land stewardship agreements
- CCOS ledger canon
- Public-facing commitments

Approval requires:

- Clear source evidence.
- Legitimate authority.
- No conflict with existing canon.
- Proper reviewer/circle involvement.
- Human approval.

## Memory Review

Memory candidates are durable context, not canon.

Approve only if:

- Useful for future work.
- Neutral.
- Sourced.
- Non-blaming.
- Specific.
- Not sensitive unless approved for memory.

## Decision Review

Decisions must be classified:

- Confirmed
- Candidate
- Unclear
- Rejected
- Needs Clarification

Canon-impacting decisions must create Canon Change Requests.

## Sensitive Review

Sensitive items should be restricted, rewritten neutrally, or routed to appropriate steward.

Do not store inflammatory interpretations as profile memory.

## Access Failures

Review failed assets after retries.

Admin may request access manually if urgent or canon-impacting.

## CCOS Ledger Drafts

Supported ledger types:

- Tension
- Proposal
- Decision
- Role
- Policy
- Resource
- Accountability

Drafts may be created automatically. Official ledger entries require approval.

3. Claude Team Project Sharing for Claude workspace access.
