# Admin-Facing Manual Summary

## Admin Role

Automation handles intake. Admins handle authority.

## Admin Surfaces

MVP admin surfaces:

1. Google Workspace Admin Console
2. Notion Living Memory Admin Dashboard
3. Claude Team Project Sharing

Optional future:

- Vercel dashboard

## Admin Review Queues

- Canon Change Requests
- Memory Review Queue
- Decision Candidates
- Sensitive Review Items
- Access Failures
- CCOS Ledger Drafts
- Unassigned Tasks
- Processing Events/System Health

## Admin Responsibilities

- Protect canon.
- Confirm decisions.
- Approve/reject memory.
- Review sensitive items.
- Handle failed access after retries.
- Review ledger drafts.
- Maintain permissions.
- Monitor system health.

## Canon Rule

Automation may suggest canon changes. Admins approve and implement canon.

## Access Failure Rule

If Roots receives a link but cannot open it, the system requests access and retries. Admin handles failures after retries.

## Weekly Admin Review

Check:

- Pending canon requests
- Memory candidates
- Decision candidates
- Sensitive flags
- Failed access
- Unassigned tasks
- Processing errors
- High severity risks
